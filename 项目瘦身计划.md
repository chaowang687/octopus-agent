# 项目瘦身计划

## 一、项目现状分析

### 1.1 代码规模统计

| 目录 | 文件数 | 代码行数 | 说明 |
|------|--------|----------|------|
| src/main/agent/ | ~100 | ~29,507 | 核心Agent代码 |
| Agent 5 Studio/Core/ | ~21 | ~7,362 | 重复的Agent代码 |
| src/renderer/src/ | ~80 | ~15,000+ | 渲染进程代码 |
| plugins/ | ~10 | ~500 | 插件代码 |

### 1.2 主要问题识别

#### 问题1：重复代码严重
- `src/main/agent/` 和 `Agent 5 Studio/Core/` 存在大量重复代码
- ConversationManager、推理引擎等核心组件重复实现
- 重复代码占比约25%

#### 问题2：测试项目过多
- projects/ 目录下包含10+个测试和示例项目
- 占用大量磁盘空间和仓库体积
- 这些项目与核心功能无关

#### 问题3：文档冗余
- 根目录下有94个markdown文档
- 多个文档内容重复或过时
- 文档组织混乱，难以维护

#### 问题4：临时文件和测试文件
- 123/ 目录（测试/临时目录）
- 简易记事本/ 目录（测试项目）
- todo-app/、sudoku-game/ 等测试项目
- 多个测试脚本（test-*.js）

## 二、瘦身目标

### 2.1 量化目标
- **代码重复率**: 从25%降至5%以下
- **项目体积**: 减少40%以上
- **文档数量**: 从94个减少到30个以下
- **测试项目**: 清理所有非核心测试项目

### 2.2 质量目标
- 代码结构更清晰
- 维护成本降低
- 构建速度提升
- 团队协作效率提高

## 三、详细瘦身方案

### 3.1 清理重复代码（优先级：高）

#### 3.1.1 删除 Agent 5 Studio/ 目录
**原因**: 与 src/main/agent/ 重复

**操作步骤**:
1. 确认 src/main/agent/ 包含所有必要功能
2. 删除整个 `Agent 5 Studio/` 目录
3. 更新所有引用路径

**预期效果**:
- 删除约7,362行重复代码
- 减少约21个文件
- 降低代码重复率至5%以下

#### 3.1.2 整合推理引擎
**当前状态**:
- OmniAgent.ts
- CognitiveEngine.ts
- EnhancedReActEngine.ts
- ThoughtTreeEngine.ts
- SelfCorrectionEngine.ts
- UnifiedReasoningEngine.ts

**整合方案**:
- 保留 UnifiedReasoningEngine.ts 作为统一入口
- 将其他引擎作为插件化实现
- 删除重复的推理引擎实现

#### 3.1.3 合并工具系统
**当前状态**:
- tools/ 目录下有多个工具文件
- tools.ts 作为工具入口

**整合方案**:
- 统一工具接口
- 将工具分类管理
- 删除未使用的工具

4. 测试和验证

#### 阶段2：清理测试项目（半天）

### 4.2 风险控制

#### 风险1：删除代码导致功能缺失
**缓解措施**:
- 在删除前进行代码分析
- 保留必要的功能
- 进行充分测试

#### 风险2：删除文档导致信息丢失
**缓解措施**:
- 仔细审查每个文档
- 保留重要信息
- 合并相似文档

#### 风险3：删除测试项目影响开发
**缓解措施**:
- 确认测试项目不影响核心功能
- 保留必要的测试代码
- 建立新的测试机制

## 七、总结

本瘦身计划旨在通过清理重复代码、删除测试项目、整理文档、优化依赖等方式，使项目更加精简、高效、易于维护。预计可以减少40%以上的项目体积，降低代码重复率至5%以下，显著提升项目的可维护性和开发效率。

执行本计划需要约3-4天时间，建议分阶段执行，每个阶段完成后进行测试和验证，确保不影响项目的核心功能。
