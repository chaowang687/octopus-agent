# Octopus Agent 本地化AI Agent桌面应用 - 测试文档

## 项目概述
Octopus Agent 是一个本地化的 AI Agent 桌面应用，集成了 AI 对话、开发工具管理、本地图库管理等功能。

**技术栈**: Electron + Vite + React + TypeScript
**包管理**: npm / yarn

## 2. 测试步骤

### 2.1 单元测试步骤

#### API密钥存储测试步骤 (UT-API-001):
1. 准备测试API密钥
2. 调用API密钥存储函数存储密钥
3. 尝试从存储位置读取数据
4. 验证存储的数据是否正确加密

#### 多模型切换测试步骤 (UT-CHAT-007):
1. 配置至少两个不同AI模型的API Key
2. 切换到第一个模型
3. 发送测试消息
4. 切换到第二个模型
5. 发送测试消息
6. 验证两个模型都能正常响应

#### 图片重命名测试步骤 (UT-GALLERY-004):
1. 准备一张测试图片
2. 在图库中选中该图片
3. 执行重命名操作，输入新名称
4. 检查文件系统中文件名是否正确修改
5. 验证文件扩展名保持不变

### 2.2 集成测试步骤

#### AI对话完整流程测试步骤 (IT-INT-002):
1. 启动应用
2. 进入设置页面
3. 配置有效的API Key
4. 保存设置
5. 进入对话页面
6. 输入测试问题
7. 点击发送按钮
8. 等待AI响应

#### 图库完整流程测试步骤 (IT-INT-004):
1. 启动应用
2. 进入图库管理页面
3. 点击导入图片按钮
4. 选择多张测试图片
5. 确认导入
6. 为不同图片添加不同标签
7. 点击某个标签进行筛选
8. 验证只显示对应标签的图片

### 2.3 端到端测试步骤

#### AI编码助手工作流测试步骤 (E2E-WF-001):
1. 启动Octopus Agent应用
2. 确认API Key已正确配置
3. 在对话输入框输入："帮我写一个简单的计算器JavaScript代码"
4. 点击发送
5. 等待AI生成完整响应
6. 检查响应内容是否包含完整的代码
7. 验证代码语法是否正确
8. 验证代码功能是否符合需求

## 4. 可能的边缘情况

### 4.1 数据边缘情况

1. **空API Key**
   - 测试：配置空字符串作为API Key
   - 预期：应用正确提示错误，不允许保存空配置

2. **超大对话历史**
   - 测试：创建一个包含上千轮对话的会话
   - 预期：应用仍然保持响应，不会卡顿或崩溃

3. **超大图片文件**
   - 测试：导入超过100MB的大图片
   - 预期：应用给出适当提示，不会崩溃，可能拒绝导入或进行压缩

4. **特殊字符文件名**
   - 测试：文件名包含特殊字符如 `/\:*?"<>|`
   - 预期：应用正确处理，不崩溃，可能自动替换特殊字符

5. **无权限的目录访问**
   - 测试：尝试访问没有读取权限的系统目录
   - 预期：应用正确提示权限错误，不崩溃

6. **磁盘空间不足**
   - 测试：磁盘空间不足时导入大量图片
   - 预期：应用正确提示空间不足，优雅失败，不损坏已有数据

### 4.2 操作边缘情况

1. **快速连续点击**
   - 测试：用户快速连续多次点击发送按钮
   - 预期：不会重复发送相同消息，或进行适当防抖处理

2. **网络断开时发送消息**
   - 测试：发送消息时网络突然断开
   - 预期：应用正确提示网络错误，用户可以重新发送

3. **应用未完全启动时进行操作**
   - 测试：应用还在启动阶段就点击各种按钮
   - 预期：应用不会崩溃，操作会在启动完成后正常处理

4. **同时打开多个窗口**
   - 测试：尝试打开多个应用窗口
   - 预期：正确处理，不允许多开或正确支持多窗口

5. **最小化到托盘后恢复**
   - 测试：应用最小化到托盘，一段时间后恢复
   - 预期：界面正常显示，所有功能正常工作

### 4.3 兼容性边缘情况

1. **macOS旧版本**
   - 测试：在macOS 10.15及更早版本运行
   - 预期：给出适当提示，或正常运行

2. **不同分辨率屏幕**
   - 测试：在4K、5K等不同分辨率屏幕运行
   - 预期：界面正常显示，缩放正确

3. **暗黑/明亮模式切换**
   - 测试：系统切换暗黑/明亮模式时
   - 预期：应用主题正确跟随系统变化
