## 🚀 智能体命令行能力

### 安全的命令执行

```typescript
class SafeCommandExecutor {
  async executeCommand(agentId: string, command: string): Promise<any> {
    // 1. 检查权限
    const { level, reason } = permissionSystem.checkCommandPermission(
      agentId,
      command
    )

    if (level === PermissionLevel.DENY) {
      throw new Error(`权限拒绝: ${reason}`)
    }

    if (level === PermissionLevel.ASK) {
      const granted = await this.askUser(`是否允许执行命令: ${command}?`)
      if (!granted) {
        throw new Error('用户拒绝了命令执行')
      }
    }

    // 2. 记录权限
    permissionSystem.recordPermission(agentId, command, level, true)

    // 3. 执行命令
    return await this.toolRegistry.execute('execute_command', { command })
  }
}
```

### 命令执行示例

```typescript
// 智能体执行命令
const result = await safeCommandExecutor.executeCommand(
  'fullstack-developer',
  'npm install'
)

// 输出
{
  success: true,
  output: 'added 142 packages in 3s',
  metadata: {
    command: 'npm install',
    executionTime: 3000,
    agentId: 'fullstack-developer'
  }
}
```

## 📊 实现效果

### 协作流程示例

**场景**：用户希望开发一个用户注册功能

1. **用户创建需求**
   - 在文库中创建需求文档
   - 描述功能需求和技术要求

2. **智能体生成计划**
   - 项目经理分析需求
   - 生成初步计划（5个步骤）
   - 保存到文库

3. **用户审查计划**
   - 查看计划详情
   - 修改步骤顺序
   - 添加补充说明

4. **智能体识别决策点**
   - 步骤2：数据库选择（PostgreSQL vs MongoDB）
   - 步骤3：认证方案（JWT vs Session）
   - 创建决策请求

5. **用户做出决策**
   - 选择 PostgreSQL
   - 选择 JWT
   - 提供选择理由

6. **智能体更新计划**
   - 根据决策更新技术栈
   - 调整依赖关系
   - 添加实施细节

7. **用户批准计划**
   - 审查完整计划
   - 确认无误
   - 开始执行

8. **智能体执行步骤**
   - 切换到 Build 模式
   - 执行步骤1：创建数据库表
   - 更新进度：✅ 已完成
   - 执行步骤2：编写API代码
   - 更新进度：🔄 执行中

9. **用户监控进度**
   - 实时查看执行进度
   - 查看执行日志
   - 暂停或取消执行

10. **任务完成**
    - 所有步骤执行完成
    - 生成执行报告
    - 记录到记忆系统
