# 部署和安装流程设计

## 1. 系统概述

部署和安装流程是macOS本地化智能体编码工具开发的重要组成部分，确保应用能够正确打包、安装和更新，为个人开发者提供流畅的使用体验。本设计文档专注于个人使用场景，涵盖应用打包、安装流程、版本管理和简单更新机制等核心方面。

## 2. 部署目标

### 2.1 核心目标

- **简化安装过程**：提供直观、快速的个人安装体验
- **确保应用安全性**：通过适当的安全措施确保应用安全
- **支持版本管理**：实现简单有效的版本控制和更新机制
- **个人部署优化**：优化应用大小和安装速度
- **跨版本兼容性**：确保应用在不同macOS版本上正常运行

### 2.2 技术目标

- **应用打包**：将应用正确打包为macOS标准格式
- **简化构建**：实现个人使用的构建流程
- **本地安装**：提供简单的本地安装方法
- **个人更新**：支持个人使用的更新机制

## 4. 部署流程

### 4.1 应用打包

#### 4.1.1 打包配置

```javascript
// electron-builder.json
{
  "appId": "com.localized-agent.coding-tool",
  "productName": "智能体编码工具",
  "copyright": "Copyright © 2026 Localized Agent",
  "mac": {
    "target": ["dmg", "zip"],
    "category": "public.app-category.developer-tools",
    "hardenedRuntime": true,
    "entitlements": "build/entitlements.mac.plist",
    "entitlementsInherit": "build/entitlements.mac.plist"
  },
  "dmg": {
    "sign": false,
    "contents": [
      {
        "x": 130,
        "y": 220
      },
      {
        "x": 410,
        "y": 220,
        "type": "link",
        "path": "/Applications"
      }
    ]
  }
}
```

#### 4.1.2 权限配置

```xml
<!-- build/entitlements.mac.plist -->
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
  <dict>
    <key>com.apple.security.app-sandbox</key>
    <true/>
    <key>com.apple.security.cs.allow-jit</key>
    <true/>
    <key>com.apple.security.cs.allow-unsigned-executable-memory</key>
    <true/>
    <key>com.apple.security.cs.allow-dyld-environment-variables</key>
    <true/>
    <key>com.apple.security.network.client</key>
    <true/>
    <key>com.apple.security.files.user-selected.read-write</key>
    <true/>
    <key>com.apple.security.automation.apple-events</key>
    <true/>
  </dict>
</plist>
```

### 4.2 个人构建流程

#### 4.2.1 构建配置

```javascript
// package.json
{
  "scripts": {
    "build": "electron-builder",
    "build:mac": "electron-builder --mac",
    "dev": "electron-vite dev",
    "test": "jest"
  }
}
```

#### 4.2.2 构建步骤

1. **安装依赖**：`npm install`
2. **开发测试**：`npm run dev`
3. **构建应用**：`npm run build`
4. **验证构建**：检查生成的安装包

## 5. 安装流程

#### 5.1.1 安装步骤

## 6. 版本管理

### 6.1 版本号规范

采用语义化版本号规范：`MAJOR.MINOR.PATCH`

- **MAJOR**：重大版本更新，可能包含不兼容的API变更
- **MINOR**：次要版本更新，添加新功能但保持向后兼容
- **PATCH**：补丁版本更新，修复bug和安全问题

### 6.2 版本管理流程

1. **版本规划**：根据个人开发需求确定版本号
2. **版本标记**：使用Git标签标记版本
3. **版本构建**：基于标签构建发布版本
4. **版本记录**：维护个人使用的版本变更日志

## 7. 更新机制

### 7.1 手动更新

#### 7.1.1 更新流程

1. **检查版本**：查看当前应用版本
2. **获取更新**：从版本控制获取最新代码
3. **重新构建**：运行构建流程生成新版本
4. **替换应用**：用新构建的应用替换旧版本

#### 7.1.2 更新验证

- **功能测试**：验证更新后的功能是否正常
- **兼容性检查**：确保与现有配置兼容
- **回滚机制**：如果更新失败，保留旧版本备份

### 7.2 版本兼容性

#### 7.2.1 配置兼容性

- 确保新版本能读取旧版本的配置文件
- 支持旧版本的插件格式
- 保持API向后兼容

## 11. 故障排除

### 11.1 常见问题

#### 11.1.1 安装问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 应用无法打开 | 权限问题 | 在系统偏好设置中允许从"任何来源"安装 |
| 安装失败 | 磁盘空间不足 | 清理磁盘空间后重新安装 |
| 应用崩溃 | 系统兼容性问题 | 检查系统要求，更新macOS版本 |
| 功能异常 | 依赖缺失 | 重新安装依赖并构建应用 |

#### 11.1.2 构建问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 构建失败 | 依赖错误 | 清除node_modules并重新安装 |
| 编译错误 | 代码问题 | 修复代码错误后重新构建 |
| 资源缺失 | 文件路径问题 | 检查资源文件路径配置 |
