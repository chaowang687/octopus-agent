# 开发路线图

## 第一阶段：内核与基础交互（4-6 周）

### 第一周：桌面端框架搭建
- Electron 项目初始化
- 悬浮胶囊窗口实现
- 全局快捷键注册
- 系统托盘图标集成

### 第二周：智能体核心接入
- 将现有的 TaskEngine、Planner、Executor 集成到 Electron 中
- 建立 Python 与 Electron 的通信机制（推荐使用 gRPC）
- 实现基础的对话界面

### 第三周到第四周：基础自动化能力
- 集成现有的 SystemAutomationService 实现键鼠操作
- 集成 ScreenPerception 实现截图和元素识别
- 接入多模态大模型（GPT-4o 或 Claude）进行屏幕理解

### 第五周到第六周：浏览器自动化
- 增强 BrowserService 支持更多操作场景
- 实现网页内容提取和表单自动填写

## 第二阶段：SOP 与工作流（6-8 周）

### 第七周到第八周：SOP 引擎开发
- 设计 SOP 配置格式和解析器
- 实现 DAG 状态机和执行引擎
- 实现断点恢复和错误处理机制

### 第九周到第十周：DSL 交互界面
- 开发可视化的流程设计器
- 支持拖拽式节点编排
- 支持条件分支和循环结构

### 第十一周到第十二周：高级流程能力
- 实现多步骤任务的进度展示和日志记录
- 实现用户确认和干预的介入机制
- 实现复杂场景下的自我纠错

## 第三阶段：多模态与优化（4-6 周）

### 第十三周到第十四周：多模态生成
- 集成 python-pptx 实现 PPT 生成
- 集成 openpyxl 实现 Excel 报表生成
- 实现文档的实时预览功能

### 第十五周到第十六周：性能优化
- 优化屏幕截图的传输效率
- 优化 Python 与 Electron 的通信延迟
- 优化内存占用和启动速度

### 第十七周：测试与发布
- 完成端到端的功能测试和性能测试
- 编写用户文档和开发者文档
- 准备发布版本
