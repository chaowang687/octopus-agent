# 工具集成模块使用说明

## 概述

工具集成模块提供了统一的接口来管理和控制各种开发工具，包括VS Code、Unity等。

## 可用工具

### 1. open_software
打开指定的开发软件

**参数：**
- `tool` (必需): 软件名称，可选值：`vscode`, `unity`, `source`, `terminal`, `browser`
- `path` (可选): 要打开的文件或项目路径
- `options` (可选): 额外的配置选项

**示例：**
```javascript
// 打开VS Code
await toolRegistry.getTool('open_software').handler({ tool: 'vscode' })

// 在VS Code中打开特定路径
await toolRegistry.getTool('open_software').handler({ 
  tool: 'vscode', 
  path: '/path/to/project' 
})
```

### 2. open_vscode
专门用于打开VS Code

**参数：**
- `path` (可选): 文件或项目路径
- `line` (可选): 跳转到指定行
- `column` (可选): 跳转到指定列

**示例：**
```javascript
// 打开VS Code
await toolRegistry.getTool('open_vscode').handler({})

// 打开特定文件
await toolRegistry.getTool('open_vscode').handler({ 
  path: '/path/to/file.ts' 
})

// 打开文件并跳转到指定位置
await toolRegistry.getTool('open_vscode').handler({ 
  path: '/path/to/file.ts',
  line: 42,
  column: 10
})
```

### 3. get_tool_status
获取开发工具的状态

**参数：**
- `tool` (可选): 特定工具名称，如果不指定则返回所有工具状态

**示例：**
```javascript
// 获取所有工具状态
await toolRegistry.getTool('get_tool_status').handler({})

// 获取VS Code状态
await toolRegistry.getTool('get_tool_status').handler({ tool: 'vscode' })
```

### 4. list_available_tools
列出所有可用的开发工具

**参数：** 无

**示例：**
```javascript
const result = await toolRegistry.getTool('list_available_tools').handler({})
console.log(result.tools)
```

## 编程接口

### VscodeIntegration

```typescript
import { VscodeIntegration } from './integration/VscodeIntegration'

const vscode = new VscodeIntegration()

// 检查VS Code是否可用
const isAvailable = vscode.isAvailable()

// 获取VS Code状态
const status = await vscode.getStatus()

// 启动VS Code实例
const instance = await vscode.startInstance({
  workingDirectory: '/path/to/project',
  debugMode: true
})

// 打开路径
await vscode.openPath('/path/to/file')

// 打开文件并跳转到指定位置
await vscode.openFile('/path/to/file.ts', 42, 10)

// 执行命令
const result = await vscode.executeCommand('--version')

// 关闭实例
await vscode.closeInstance(instance.id)

// 关闭所有实例
await vscode.closeAllInstances()
```

### ToolCallCenter

```typescript
import { toolCallCenter } from './integration/ToolCallCenter'

// 列出可用工具
const tools = toolCallCenter.listAvailableTools()

// 获取工具状态
const status = await toolCallCenter.getToolStatus('vscode')

// 获取所有工具状态
const allStatuses = await toolCallCenter.getAllToolStatuses()

// 打开工具
await toolCallCenter.openTool('vscode', { debugMode: true })

// 使用工具打开路径
await toolCallCenter.openPath('vscode', '/path/to/project')

// 执行工具命令
const result = await toolCallCenter.executeCommand('vscode', '--version')

// 自动选择工具
const tool = await toolCallCenter.autoSelectTool('write some code')
```

## 使用场景

### 场景1：在任务执行过程中打开VS Code

```typescript
// 在TaskEngine中
const tool = await toolCallCenter.autoSelectTool('edit the code')
await toolCallCenter.openPath(tool, projectPath)
```

### 场景2：检查开发环境

```typescript
// 检查所有开发工具是否可用
const allStatuses = await toolCallCenter.getAllToolStatuses()
for (const [toolName, status] of allStatuses) {
  console.log(`${toolName}: ${status.available ? 'Available' : 'Not Available'}`)
}
```

### 场景3：打开文件进行编辑

```typescript
// 打开特定文件并跳转到错误行
await toolCallCenter.openPath('vscode', filePath)
// 或者
const vscode = new VscodeIntegration()
await vscode.openFile(filePath, errorLine, errorColumn)
```

```bash
npm test -- tests/integration/ToolIntegration.test.ts
```

## 扩展

要添加新的工具集成，请：

1. 在 `ToolTypes.ts` 中定义新的接口和类型
2. 创建新的集成类实现 `ToolIntegration` 接口
3. 在 `ToolCallCenter` 中注册新工具
4. 在 `OpenTool.ts` 中注册相应的工具函数
5. 添加测试用例

## 注意事项

- 确保目标工具已安装在系统中
- 某些功能可能需要工具处于特定状态才能正常工作
- 使用完毕后记得关闭工具实例以释放资源
