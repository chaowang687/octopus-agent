## 7. 工作流程

### 流程图

```
UI设计 → 需求理解 → 架构设计 → 代码实现 → 测试调试 → 优化
    ↓         ↓          ↓          ↓          ↓        ↓
  理解设计   分析需求   设计结构   编写代码   测试功能   优化性能
```

### 详细步骤

1. **需求理解**
   - 读取UI设计文档
   - 理解功能需求
   - 识别技术要点
   - 确定技术栈

2. **架构设计**
   - 设计系统架构
   - 规划模块结构
   - 设计数据流
   - 确定接口规范

3. **代码实现**
   - 创建项目结构
   - 编写核心代码
   - 实现业务逻辑
   - 集成第三方库

4. **测试调试**
   - 编写测试用例
   - 执行测试
   - 修复bug
   - 验证功能

5. **优化**
   - 优化性能
   - 优化代码质量
   - 优化用户体验
   - 文档完善

## 9. 错误处理

### 常见错误

| 错误类型 | 错误信息 | 处理方式 |
|---------|---------|---------|
| BuildError | 构建失败 | 检查依赖，修复错误 |
| RuntimeError | 运行时错误 | 调试代码，修复bug |
| PermissionError | 权限错误 | 请求权限或调整路径 |
| DependencyError | 依赖错误 | 安装或更新依赖 |

### 错误恢复策略

1. **构建失败**
   - 检查构建日志
   - 识别错误原因
   - 修复代码或配置
   - 重新构建

2. **运行时错误**
   - 查看错误堆栈
   - 定位错误代码
   - 修复逻辑错误
   - 重新测试

3. **权限错误**
   - 检查文件权限
   - 请求用户权限
   - 调整文件路径
   - 重新尝试

4. **依赖错误**
   - 检查package.json
   - 安装缺失依赖
   - 更新依赖版本
   - 清理缓存

### C. FAQ

**Q1: 开发工程师智能体能执行哪些命令？**
A: 可以执行npm install、npm run dev、npm run build、npm test等命令，其他命令需要用户确认。

**Q2: 开发工程师智能体能修改哪些文件？**
A: 可以修改.ts、.tsx、.js、.jsx、.json、.css、.scss、.html、.md等文件，但不能修改敏感文件。

**Q3: 开发工程师智能体如何处理构建错误？**
A: 检查构建日志，识别错误原因，修复代码或配置，然后重新构建。

**Q4: 开发工程师智能体的输出为什么这么简单？**
A: 为了让用户能理解开发进度，使用日常语言描述技术实现，避免使用技术术语。
